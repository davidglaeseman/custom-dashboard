# Update Pull Request Title and Description

## Usage
Trigger this skill whenever you need to update an existing PR by running `/update-pr <pr_number>`.

## Workflow
1. **Analyze Full Diff:** Use `git diff` against the base branch to get a comprehensive view of all changes in the PR.
2. **Determine Title:** Generate a PR title using Conventional Commits (e.g., `feat: add user authentication endpoint`).
3. **Draft Description:** Create a structured description using the following markdown format:
    - **What Changed:** A bulleted list of key modifications.
    - **Why:** The core purpose or problem solved.
    - **Testing Plan:** Brief steps on how to verify the changes.
4. **Update GitHub:** Use the GitHub CLI (`gh pr edit <pr_number> --title "<title>" --body "<body>"`) to apply the changes.

## Rules
- **Do not** add any AI attribution (e.g., "Generated by Claude") in the PR description.
- Ensure the title and description are strictly professional, clear, and orientation-focused for reviewers.
- Always review the entire diff context rather than just the most recent commits.
